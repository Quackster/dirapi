3. CHUNK-TYPE DISPATCH TABLE (Vtable-like Structure)
================================================================================

FUN_680031e0 at 0x680031e0 is the chunk-type dispatcher.
It maps FourCC values to function pointer tables:

  0x52494658 ("RIFX") -> PTR_FUN_68188084  (RIFX container handler)
  0x6d6d6170 ("mmap") -> PTR_FUN_68188070  (memory map handler)
  0x6a756e6b ("junk") -> PTR_FUN_6818805c  (junk/padding handler)
  default            -> PTR_FUN_68188048  (generic chunk handler)

Each table entry contains function pointers used by Ordinal_16 (ReadChunk):
  [0] = Create/allocate handler  (called to create chunk data in memory)
  [1] = (unknown, possibly validate)
  [2] = GetSize handler          (called by FUN_68003a00)
  [3] = Read/decode handler      (called to parse chunk data from stream)

These tables are at addresses in the .data section (0x68188048-0x68188084+).

The FGDM (0x4647444d) and FGDC (0x46474443) magic values represent Director
movie file formats and trigger special code paths throughout the ordinals:
  - FGDM = likely "MDGF" = Director Movie/Media File (big-endian FourCC)
  - FGDC = likely "CDGF" = Director Cast File (big-endian FourCC)

When these magic values are detected in the file object's first dword,
alternate code paths handle Director-specific container features like
multi-cast libraries and embedded resources.

9. SUMMARY
================================================================================

dirapi.dll is the core Director file I/O engine. Ordinals 10-23 form the
primary IFF/RIFF container manipulation API ("MIO" - Media I/O):

  Open/Close:     Ordinal_10 (open), Ordinal_11 (close), Ordinal_12 (set file)
  Read/Parse:     Ordinal_13 (read all), Ordinal_16 (read chunk)
  Write/Save:     Ordinal_14 (save), Ordinal_19 (add chunk), Ordinal_20 (set data)
  Header:         Ordinal_21 (write header), Ordinal_22 (update offset),
                  Ordinal_23 (compute size)
  Query:          Ordinal_15 (get error), Ordinal_17 (find chunk),
                  Ordinal_18 (get chunk info)

The DLL uses a chunk-type dispatch table at FUN_680031e0 that maps FourCC
codes (RIFX, mmap, junk) to handler function tables, similar to a vtable.

It imports handle management and synchronization primitives via thunked
ordinals from the MOA runtime (likely IML32.DLL).

The file format magic values FGDM and FGDC trigger Director-specific code
paths for movie files and cast library files respectively.
